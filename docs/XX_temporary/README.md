# 一時ドキュメント置き場（XX_temporary）

このフォルダは、**一時的なメモ、レビュー、検討資料**を置くための場所である。

## 目的

- ドキュメント化する前の**一時的なメモ**を保存
- **レビュー資料**や**検討中の内容**を一時的に保管
- 後で評価し、適切なVモデル構造のフォルダに分類・分割・統合する前の**ステージングエリア**

## 使い方

1. **一時的なメモやレビューを作成する際**:
   - このフォルダに直接ファイルを作成する
   - ファイル名はケバブケース（`{theme}-{type}.md`）を推奨
   - 例: `postgresql-migration-review.md`, `new-feature-consideration.md`

2. **定期的な評価と整理**:
   - 定期的に（例: スプリント終了時）このフォルダの内容をレビューする
   - 各ファイルを以下のいずれかに分類する:
     - **分類**: Vモデル構造の適切なフォルダ
       （`00_planning/`, `10_requirements/`, `20_architecture/`, など）に移動
     - **分割**: 複数のファイルに分割して各フォルダに配置
     - **統合**: 既存のドキュメントに統合
     - **削除**: 不要になった場合は削除

3. **ファイルの命名規則**:
   - ケバブケース（`kebab-case`）を使用
   - 内容が分かるような名前を付ける
   - 例:
     - `feature-x-review.md`
     - `postgresql-performance-notes.md`
     - `adr-candidate-0008.md`

## 注意事項

- **このフォルダの内容は一時的なもの**として扱うこと
- **定期的に整理**し、長期間放置しないこと
- **重要な情報は早めに適切な場所に移動**すること
- このフォルダ内のファイルは、`docs/README.md`のインデックスには含めない

## 整理の判断基準

### 分類先の決定

- **要件・企画関連**: `00_planning/` または `10_requirements/`
- **アーキテクチャ決定**: `20_architecture/22_adrs/`
- **設計関連**: `30_design_basic/` または `40_design_detailed/`
- **実装ガイド**: `50_implementation/51_guides/` または `50_implementation/52_procedures/`
- **テスト関連**: `60_testing/`
- **運用関連**: `90_operations/`

### 分割が必要な場合

- 1つのファイルに複数のフェーズ（要件と実装など）が混在している場合
- ファイルサイズが1,000行を超える場合
- テーマが異なる複数の内容が含まれている場合

### 統合が必要な場合

- 既存のドキュメントと内容が重複している場合
- 既存のドキュメントの補足情報として追加できる場合
- 同じテーマの既存ドキュメントがある場合
